# План миграции компонентов из freecode.academy в agent

## Исходные данные

### Базовый проект: `/agent`
- **Стек**: Next.js 16, React 18, Express 4, Apollo Server 5, Pothos GraphQL, Prisma 6, PostgreSQL
- **Особенности**: 
  - AI Agent с системой памяти (11 типов MindLogs)
  - n8n интеграция для workflow
  - JWT аутентификация
  - Knowledge Base система (факты, концепты, конфликты)
  - Reflex + Reaction система

### Источник компонентов: `/freecode.academy`
- **Стек**: Next.js 12, React 18, Express 4, Apollo Server 4, Nexus, Prisma 4, MySQL
- **Ключевые компоненты**:
  - CodeChallenge система (задания с тестами)
  - Monaco Editor интеграция
  - Система проверки кода (useExecuteChallenge)
  - LearnStrategy (треки обучения)
  - Babel транспиляция + loop protection

## Этап 3: Миграция GraphQL схемы

### 3.1 Создать Pothos типы для заданий
- [ ] Создать `/agent/server/schema/types/Challenge.ts`
  ```typescript
  import { builder } from '../builder';
  
  builder.prismaObject('Challenge', {
    fields: (t) => ({
      id: t.exposeID('id'),
      name: t.exposeString('name'),
      description: t.exposeString('description', { nullable: true }),
      instructions: t.exposeString('instructions', { nullable: true }),
      challengeType: t.expose('challengeType', { type: ChallengeTypeEnum }),
      difficulty: t.exposeInt('difficulty'),
      steps: t.relation('steps'),
      completions: t.relation('Completions'),
      createdBy: t.relation('CreatedBy'),
    }),
  });
  ```

- [ ] Создать queries
  ```typescript
  builder.queryFields((t) => ({
    challenge: t.prismaField({
      type: 'Challenge',
      args: { id: t.arg.string({ required: true }) },
      resolve: (query, root, args, ctx) =>
        ctx.prisma.challenge.findUniqueOrThrow({
          ...query,
          where: { id: args.id },
        }),
    }),
    
    challenges: t.prismaField({
      type: ['Challenge'],
      args: {
        type: t.arg({ type: ChallengeTypeEnum, required: false }),
        difficulty: t.arg.int({ required: false }),
      },
      resolve: (query, root, args, ctx) =>
        ctx.prisma.challenge.findMany({
          ...query,
          where: {
            challengeType: args.type,
            difficulty: args.difficulty,
          },
        }),
    }),
  }));
  ```

- [ ] Создать mutations
  ```typescript
  builder.mutationFields((t) => ({
    submitChallengeStep: t.prismaField({
      type: 'ChallengeCompletion',
      args: {
        challengeId: t.arg.string({ required: true }),
        stepNumber: t.arg.int({ required: true }),
        answer: t.arg({ type: 'JSON', required: true }),
      },
      resolve: async (query, root, args, ctx) => {
        // Логика проверки шага
        const validation = await validateStep(
          args.challengeId,
          args.stepNumber,
          args.answer
        );
        
        // Обновить прогресс
        return ctx.prisma.challengeCompletion.update({
          ...query,
          where: { id: completion.id },
          data: {
            stepsCompleted: { push: args.stepNumber },
            currentStep: args.stepNumber + 1,
          },
        });
      },
    }),
  }));
  ```

## Заметки по миграции

### Различия стеков
- **Nexus → Pothos**: Pothos более type-safe, но синтаксис отличается
- **MySQL → PostgreSQL**: Нужно адаптировать типы данных (особенно JSON)
- **Prisma 4 → Prisma 6**: Обновленный синтаксис, новые фичи

### Что можно переиспользовать как есть
- Monaco Editor интеграция (с минимальными изменениями)
- Логика Babel транспиляции
- Loop protection механизм
- UI компоненты (styled-components)

### Что нужно переписать
- GraphQL схема (Nexus → Pothos)
- Модели БД (адаптация под PostgreSQL)
- Серверная логика проверки (адаптация под новую структуру)

### Преимущества базы agent
- Более современный стек (Next.js 16, Prisma 6)
- Встроенная AI система (n8n + OpenAI)
- Система памяти (MindLog)
- Knowledge Base для хранения знаний
- Reflex система для автоматизации
