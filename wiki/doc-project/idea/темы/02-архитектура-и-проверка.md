# Архитектура системы и автоматическая проверка

## Общая архитектура

Для каждого задания:

1. **Формальное описание правил** (machine-readable)
2. **Модель "правильного решения"**
3. **Набор проверок (checks)**, которые:
   - независимы
   - пошаговые
   - дают фидбек

Пример структуры:
```json
[
  { "rule": "таблица_есть", "weight": 1 },
  { "rule": "значения_в_ячейках", "weight": 3 },
  { "rule": "диаграмма_тип", "weight": 2 },
  { "rule": "диаграмма_источник", "weight": 2 }
]
```

## Принципы проверки

### Один скрипт — все устройства

**check_engine (сервер)**
- Принимает абстрактную модель решения
- Проверка всегда одна
- Меняется только UI-слой

### Пошаговая проверка (ключевая фишка)

Не "прошло / не прошло", а:

✔ Таблица создана
✔ Формулы есть
❌ Диаграмма использует неверный диапазон
💡 Подсказка: "диаграмма должна строиться по столбцам A и B"

**Это убьёт репетиторов (в хорошем смысле)**

## Объяснение ошибок

### Каждая проверка = один человеческий текст

Структура правила:
```json
{
  "id": "chart_data_range",
  "error_message": "Диаграмма построена по неверному диапазону данных",
  "hint": "Выдели столбцы A и B со значениями и построй диаграмму заново",
  "severity": "error"
}
```

❗ Текст не технический, а как будто проверяет учитель.

### Многоуровневые ошибки

Как в хороших IDE:

- ❌ **Ошибка** — дальше нельзя
- ⚠️ **Предупреждение** — засчитают, но не идеально
- 💡 **Совет** — улучшение

### Словарь ошибок (отдельный слой)

```json
{
  "E_CELL_VALUE": {
    "title": "Неверное значение в ячейке",
    "student_text": "Значение в ячейке {cell} не совпадает с ожидаемым",
    "hint": "Проверь формулу в этой ячейке"
  }
}
```

Преимущества:
- Легко меняешь тексты
- Добавляешь версии "попроще"
- Адаптируешь под 9 класс

## Визуализация ошибок

### Привязка ошибки к объекту

После загрузки файла:
- Парсишь файл
- Строишь виртуальное представление
- Подсвечиваешь конкретные ячейки / диаграммы

Пример фидбека:
```
❌ Диаграмма построена неправильно
📍 Проблема связана с диапазоном A2:C6
```

Показывается через:
- Цветную рамку
- Стрелку
- Наведение
