# Пример: Исполнитель Вычислитель

## Условие задачи

У исполнителя Вычислитель две команды, которым присвоены номера:

1. умножь на 4
2. вычти b

(b — неизвестное натуральное число)

Первая из них увеличивает число на экране в 4 раза, вторая уменьшает его на b. 

Известно, что программа **21122** переводит число **4** в число **28**.

**Определите значение b.**

## Как решается (коротко)

Программа 21122 читается слева направо.

Старт: 4

1️⃣ команда 2: 4 − b
2️⃣ команда 1: (4 − b) · 4
3️⃣ команда 1: 4(4 − b) · 4 = 16(4 − b)
4️⃣ команда 2: 16(4 − b) − b
5️⃣ команда 2: 16(4 − b) − 2b

По условию:
```
16(4 − b) − 2b = 28
64 − 16b − 2b = 28
64 − 18b = 28
18b = 36
b = 2
```

## КАК ЭТО УЧИТЬ

### ❌ Как учат обычно (плохо)
- «Подставляй»
- «Догадайся»
- «Попробуй b = 1, 2, 3…»

Это анти-алгоритм. Он не масштабируется.

### ✅ Правильная модель обучения

👉 Исполнитель = чёрный ящик
👉 Команда = функция
👉 Программа = композиция функций

И ты учишь не считать, а составлять выражение.

## Учебный алгоритм (который реально работает)

### Шаг 1. Запрещаем считать числа

Да, прямо запрещаем.

Вводим переменную:
- x — текущее число

Команды:
- 1: x → 4x
- 2: x → x − b

### Шаг 2. Проходим программу как строку

Не считаем, а переписываем:

| Команда | Было | Стало |
|---------|------|-------|
| старт | — | 4 |
| 2 | 4 | 4 − b |
| 1 | 4 − b | 4(4 − b) |
| 1 | 4(4 − b) | 16(4 − b) |
| 2 | 16(4 − b) | 16(4 − b) − b |
| 2 | 16(4 − b) − b | 16(4 − b) − 2b |

👉 Ученик видит рост выражения, а не теряется в вычислениях.

### Шаг 3. Только в конце — уравнение

Теперь:
```
16(4 − b) − 2b = 28
```

Решаем одно уравнение, а не 5 шагов подряд.

## Как это автоматизировать в обучающей системе

### Интерфейс (идеально для мобилки)

Не поле "ответ", а пошаговый конструктор выражения.

Ученик:
1. Видит текущий результат: **4**
2. Нажимает команду **2** → система показывает: **4 − b**
3. Нажимает **1** → **4(4 − b)**
4. ...

👉 нельзя ошибиться в арифметике
👉 ошибка = логическая, а не счётная

### Проверки (скрипт)

Для каждого шага:
- структура выражения
- правильная замена
- количество b

Например:
```json
{
  "step": 4,
  "mustContain": ["16", "(4-b)", "-b"]
}
```

### Подсказки (умные, а не "решение")

❌ «Неверно»
✅ «После команды 2 выражение должно уменьшаться на b»

❌ «Ошибка»
✅ «Ты умножил только 4, а нужно умножать ВСЁ выражение»

## Две версии заданий

### 1️⃣ Тренировочная
- пошаговая
- видно выражение
- подсказки

### 2️⃣ Экзаменационная
- только начало
- только программа
- только конечный результат

**Если первую научился решать — вторую решит на бумаге.**

## Что ученик реально усваивает

- что программа — это последовательность функций
- что неизвестное можно тащить до конца
- что думать выгоднее, чем подбирать
