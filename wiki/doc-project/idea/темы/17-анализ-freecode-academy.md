// Babel транспилирует JSX в JavaScript

## Что важно для ThinkSteps.academy

### ✅ Взять из freecode.academy

#### 1. Структура "Теория + Практика"
```
Левая панель:
- 📖 Объяснение метода
- 💡 Примеры
- ⚠️ Важные моменты

Правая панель:
- ✍️ Интерактивное задание
- 🤖 Автопроверка
- 📊 Результат
```

#### 2. Немедленная обратная связь
- Кнопка "Проверить" / "Run Tests"
- Мгновенный результат
- Конкретные ошибки

#### 3. Блокировка до выполнения
- Не можешь идти дальше, пока не решишь
- Мотивирует довести до конца

#### 4. Стартовый код
- Не пустой редактор
- Уже есть структура
- Нужно только исправить/дополнить

### ❌ Не брать (не подходит для ОГЭ)

#### 1. Только код в редакторе
- Для ОГЭ нужны разные типы заданий
- Не только программирование

#### 2. Одна проверка на всё задание
- Для ОГЭ нужна **пошаговая** проверка
- Каждый шаг алгоритма отдельно

## Технические детали

### Как реализовать пошаговую проверку

```javascript
// Структура задания
const task = {
  id: "executor-calculator",
  steps: [
    {
      number: 1,
      title: "Запиши команды как функции",
      type: "input",
      fields: [
        { name: "command1", expected: "4x" },
        { name: "command2", expected: "x-b" }
      ],
      validation: (answer) => {
        // Проверка шага 1
        return {
          passed: true/false,
          error: "...",
          hint: "..."
        }
      }
    },
    {
      number: 2,
      title: "Пройди программу",
      type: "expression_builder",
      // ...
    }
  ]
}

// API для проверки шага
POST /api/tasks/:taskId/check-step
{
  "stepNumber": 1,
  "answer": {
    "command1": "4x",
    "command2": "x-b"
  }
}

// Ответ
{
  "passed": true,
  "nextStep": 2,
  "feedback": "Отлично! Переходи к следующему шагу"
}
```

### Типы интерактивных элементов

#### 1. Текстовый ввод
```jsx
<Input 
  placeholder="Введи выражение"
  validate={(value) => checkExpression(value)}
/>
```

#### 2. Конструктор выражений
```jsx
<ExpressionBuilder
  startValue="4"
  commands={["×4", "-b"]}
  onBuild={(expression) => checkStep(expression)}
/>
```

#### 3. Выбор из списка
```jsx
<Select>
  <option>60-69</option>
  <option>70-79</option>
  <option>80-89</option>
</Select>
```

#### 4. Drag & Drop
```jsx
<DragDropArea
  items={["команда 1", "команда 2"]}
  onDrop={(sequence) => checkSequence(sequence)}
/>
```

## Сравнение подходов

| Аспект | freecode.academy | ThinkSteps.academy |
|--------|------------------|-------------------|
| **Проверка** | Одна на всё задание | Пошаговая |
| **Тип заданий** | Только код | Разные типы (таблицы, графы, код) |
| **Обратная связь** | "Тест пройден/не пройден" | Конкретная ошибка на конкретном шаге |
| **Подсказки** | Нет | Есть на каждом шаге |
| **Визуализация** | Результат кода | Визуализация процесса решения |
| **Прогресс** | Линейный | По трекам с разблокировкой |

## Ключевые выводы

### Что делает freecode.academy хорошим

1. **Немедленная обратная связь** - не нужно ждать
2. **Практика сразу после теории** - закрепление материала
3. **Нельзя пропустить** - гарантия усвоения
4. **Стартовый код** - снижает порог входа

### Что нужно улучшить для ОГЭ

1. **Пошаговая проверка** - не только финальный результат
2. **Разные типы заданий** - не только код
3. **Умные подсказки** - помощь на каждом шаге
4. **Визуализация процесса** - показать ход мышления
5. **Адаптивность** - под уровень ученика
