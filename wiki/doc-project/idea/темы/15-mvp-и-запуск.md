# MVP и стратегия запуска

## Идея проекта (как она есть на самом деле)

Ты делаешь не публичную платформу для всех, а:

- закрытое учебное пространство
- для своих учеников
- с ЛК
- с заданиями формата ОГЭ
- с автопроверкой шагов
- и чужими видео ТОЛЬКО через iframe

Это по сути:
> **личный цифровой класс + тренажёр мышления**

И это 🔥.

## Почему iframe + ссылки — это ПРАВИЛЬНО

### 1️⃣ Юридически безопасно

Ты:
- не хранишь видео
- не распространяешь контент
- не выдаёшь его за свой
- используешь стандартный механизм YouTube

iframe = ты просто «окно» на чужой ресурс.

👉 Это 100% ок, даже если проект вырастет.

### 2️⃣ Методически логично

Ты не конкурируешь с учителями за:
- объяснение темы

Ты закрываешь другое:
- практику
- алгоритмы
- проверку шагов
- обратную связь

**Видео = ввод**
**ThinkSteps = работа мозгом**

### 3️⃣ Идеально для старта

Тебе не нужно:
- записывать 100 видео
- заниматься продакшеном
- тратить месяцы

Ты:
- берёшь лучшие объяснения
- добавляешь свою ценность
- тестируешь методику

**Это MVP в лучшем виде.**

## Архитектура проекта (простая и рабочая)

### Для ученика

1. Видео (iframe)
2. Краткий алгоритм решения
3. Задание
4. Пошаговая проверка
5. Ошибки → подсказки
6. Статистика прогресса

### Для тебя

- ты выбираешь видео
- ты управляешь заданиями
- ты видишь шаги учеников
- ты понимаешь, где именно они ломаются

**Это огромный плюс по сравнению с обычным репетиторством.**

## Почему «только по ссылке ученикам» — это умно

### Пока:
- нет публичного маркетинга
- нет рисков с ФИПИ
- нет лишнего внимания

### Это:
- закрытый клуб
- образовательная среда
- рабочий инструмент

### Позже ты сможешь:
- открыть часть контента
- оставить часть по приглашению
- или масштабироваться

## Как это правильно подать

❌ **Не так:**
> «У нас есть видео других учителей»

✅ **Так:**
> «Мы используем проверенные открытые образовательные материалы, а основной упор делаем на практику и пошаговую проверку решений»

## Описание проекта одной фразой

> ThinkSteps.academy — это закрытая учебная платформа для учеников, где мы учимся решать задачи ОГЭ шаг за шагом, а не просто подбирать ответы.

## MVP: что делать в первую очередь

### Фаза 1: Минимальный функционал (2-4 недели)

**Обязательно:**
1. Регистрация и авторизация (JWT)
2. ЛК ученика (базовый)
3. Один тип задания (исполнители - самый простой для старта)
4. Пошаговая проверка для этого типа
5. Встраивание видео через iframe
6. Базовая статистика (сколько задач решено)

**НЕ делать:**
- Красивый дизайн (достаточно чистого и понятного)
- Множество типов заданий
- Сложную аналитику
- Мобильное приложение (только адаптивный веб)
- Платежи

### Фаза 2: Расширение (1-2 месяца)

**Добавить:**
1. Ещё 2-3 типа заданий (таблицы, диаграммы)
2. Улучшенные подсказки
3. Режим "как на экзамене"
4. Детальная статистика прогресса
5. Возможность преподавателю видеть прогресс учеников

### Фаза 3: Полировка (по мере роста)

**Добавить:**
1. Все типы заданий ОГЭ
2. Собственные видео
3. Адаптивные подсказки (в зависимости от уровня)
4. Система достижений
5. Запись на занятия через платформу

## Метрики успеха MVP

### Для учеников:
- Средний процент правильных шагов > 70%
- Время на задание уменьшается от попытки к попытке
- Ученики возвращаются регулярно (retention)

### Для преподавателя:
- Видно, где ученики спотыкаются
- Меньше времени на объяснение одного и того же
- Больше времени на индивидуальную работу

### Для бизнеса:
- Ученики записываются на занятия через платформу
- Рекомендуют другим
- Готовы платить за доступ

## Риски и как их минимизировать

### Риск 1: Технические сложности
**Решение:** Начать с одного типа заданий, использовать готовые решения

### Риск 2: Контент
**Решение:** iframe для видео, свои формулировки заданий

### Риск 3: Юридические вопросы
**Решение:** Следовать правилам по ФИПИ и авторским правам

### Риск 4: Ученики не используют
**Решение:** Интегрировать в процесс обучения, делать обязательным

## Следующие шаги

1. ✅ Концепция готова
2. ✅ Темы структурированы
3. 🔄 Выбрать первый тип задания для реализации
4. 🔄 Настроить базовую инфраструктуру
5. 🔄 Создать первое задание с проверкой
6. 🔄 Протестировать на 2-3 учениках
7. 🔄 Собрать обратную связь
8. 🔄 Итерировать

## Критерии готовности к запуску

MVP готов, когда:
- ✅ Ученик может зарегистрироваться
- ✅ Посмотреть видео
- ✅ Решить задание пошагово
- ✅ Получить обратную связь
- ✅ Увидеть свой прогресс
- ✅ Преподаватель видит статистику

**Всё остальное — потом.**
