# Мобильная версия и UX

## Главный принцип для мобилки

**Ученик не делает файл. Файл делает система.**

Ученик:
- тыкает
- вводит
- перетаскивает
- выбирает

Система:
- хранит состояние
- формирует "виртуальный ODS"
- прогоняет проверки

## Архитектура «один скрипт — все устройства»

### 1️⃣ Одна логика проверки

**check_engine (сервер)**

Он:
- принимает абстрактную модель решения
- проверяет правила
- возвращает результат

### 2️⃣ Адаптивный UI-слой

То есть:
- скрипты проверок не меняются
- меняется только UI-слой

## Что реально можно делать на телефоне

### 1️⃣ Таблицы — МОЖНО, если сделать правильно

Не «Excel», а табличный конструктор.

**На мобилке:**
- Тап по ячейке → появляется:
  - число
  - формула (упрощённая)
  - выбрать из списка функций

- Диапазон выделяется:
  - долгим тапом
  - или "выбери от A2 до B6"

❗ Никакого drag'а мышью.

**Проверка:**
- такая же, как на ПК
- просто данные уже у тебя в JSON

### 2️⃣ Диаграммы — вообще идеально для мобилки

Парадоксально, но факт.

**UI:**
- Выбрал тип: круговая / столбчатая
- Выбрал данные (чекбоксы): ☑ столбец A, ☑ столбец B
- Нажал «Построить»

**Проверка:**
- тип диаграммы
- источник данных
- позиция (логическая, не пиксели)

**На мобилке даже проще, чем на ПК.**

### 3️⃣ Презентации — только шаблоны

❌ Свободный дизайн → ад
✅ Шаблон + правила

**Что даёшь ученику:**
- 3–4 готовых макета слайда
- Фиксированные зоны: заголовок, текст, изображение

**Ученик:**
- вводит текст
- выбирает картинку
- меняет местами блоки

**Проверка:**
- заполнены ли нужные зоны
- объём текста
- наличие изображения

Для ОГЭ — более чем достаточно.

### 4️⃣ Текстовые задания

На мобилке:
- не "пиши как хочешь"
- а **выбери стиль**

**Пример:**
- стиль: Обычный текст
- стиль: Заголовок
- стиль: Таблица 3×4

**Шрифт:**
- задаётся стилем
- ученик его не выбирает

➡️ Проверка становится тривиальной.

### 5️⃣ Python — мобилка вообще не проблема

Можно использовать:
- встроенный редактор кода
- виртуальную клавиатуру с символами
- автодополнение

## Особенности для 1 части ОГЭ

### Проблема

На экзамене не будет компьютера — только бумага.
Рисовать графы удобно на бумаге, а с телефона нет.

### Решение

🎯 **Тренировать нужно не рисование, а принятие решений**

Ученик выполняет МЫСЛЕННЫЕ шаги алгоритма, а система проверяет, что шаги правильные.

Не «нарисуй», а:
- выбери
- отметь
- пройди
- зафиксируй

## UX-принципы для мобилки

### 1. Большие кнопки
- Минимум 44×44 пикселя
- Достаточное расстояние между элементами

### 2. Вертикальная прокрутка
- Избегать горизонтальной прокрутки
- Контент адаптируется под ширину экрана

### 3. Минимум ввода текста
- Максимум выбора из списков
- Автодополнение где возможно

### 4. Пошаговость
- Один шаг — один экран
- Прогресс-бар показывает, где ты находишься

### 5. Быстрая обратная связь
- Мгновенная реакция на действия
- Визуальное подтверждение выбора

## Стратегия платформы

### До экзамена
- интерактив
- клики
- шаги
- алгоритмы

### Перед экзаменом
Отдельный режим: «как на экзамене»
- без подсказок
- без визуализации
- только вопрос → ответ

## Вывод

❌ Пытаться «перенести тетрадь в телефон» — ошибка
✅ Нужно перенести мышление в интерфейс

И тогда:
- мобилка ≠ проблема
- 1 часть ОГЭ автоматизируема
- подготовка становится сильнее, чем «просто писать»
